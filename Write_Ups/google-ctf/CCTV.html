Solution Process

Opening the link took me to a website asking for a password. I attempted to login by entering a random password, however an alert appeared saying the password is wrong.

Seeing that the challenge is said to be a reverse engineering one, I decided to view the source code by pressing “Ctrl + U”. Looking through the code, I found a checkPassword function, which seemed to be checking if an entered password matched the correct one. 

const checkPassword = () => {
  const v = document.getElementById("password").value;
  const p = Array.from(v).map(a => 0xCafe + a.charCodeAt(0));
if(p[0] === 52037 && 
	p[6] === 52081 && 
	p[5] === 52063 && 
	p[1] === 52077 && 
	p[9] === 52077 && 
	p[10] === 52080 && 
	p[4] === 52046 && 
	p[3] === 52066 && 
	p[8] === 52085 && 
	p[7] === 52081 && 
	p[2] === 52077 && 
	p[11] === 52066) { 
window.location.replace(v + ".html"); 
} else { 
	alert("Wrong password!"); 
	} 
}

Starting from analyzing the script; it converts the input password into an array, then each character is converted into its ASCII value and added to 0xCafe (hexadecimal for 51966). So, from here, to find the password, we just have to reverse the steps from the JavaScript code.

To do so, I made a small python script which when executed printed the CCTV panel’s password; “GoodPassword”.

#create an array with items in order from 0-12 from source code function
pass_arr = [52037, 52077, 52077, 52066, 52046, 
52063, 52081, 52081, 52085, 52077, 52080, 52066]

#convert back to character by subtracting the hex 
for char in pass_arr:
    print(chr(char-0xcafe), end="")

Upon entering this, the page was redirected, and the flag was revealed at the bottom to be “CTF{IJustHopeThisIsNotOnShodan}”.
